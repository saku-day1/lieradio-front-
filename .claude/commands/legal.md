公開前・マージ前の変更差分（`git diff main...HEAD`）または新しく追加する機能・ページを対象に、以下の6項目を順番にチェックしてください。

各項目は「✅ 問題なし」「⚠️ 要確認」「❌ 公開不可」の3段階で評価し、問題がある場合は該当箇所と対処方法を示すこと。

---

## 1. 公式導線を阻害していないか

ラジオ・動画・公式サービスへのアクセスを妨げる設計になっていないか確認する。

チェック観点：
- YouTube 動画へのリンクが正しい URL を指しているか（短縮・リダイレクトで本来の動画に到達できるか）
- 「YouTube で見る」「公式サイトへ」などの外部リンクが、このサイト内で閲覧完結するような導線に置き換わっていないか
- 動画プレイヤーを埋め込んでいる場合、YouTube の埋め込みポリシーに準拠しているか（`allowfullscreen` 等）
- ファンサイトであることが明示されており、公式と誤認される表現・デザインになっていないか
- ユーザーを公式コンテンツへ誘導する動線（番組ページ・配信サービスへのリンク等）が維持されているか

## 2. 本文転載になっていないか

番組・放送の内容テキストを転載していないか確認する。

チェック観点：
- ラジオ番組のトーク内容・台本・セリフを文字起こし・要約してコミットしていないか
- `data/episodeMeta.json` や Google Sheets に「放送内容のまとめ」テキストが入っていないか
- エピソードの「概要」として YouTube 説明文をそのまま長文で表示していないか（タイトルや短い補足は可）
- コーナー名・楽曲名・イベント名は「索引情報」として許容されるが、その説明テキストを転載していないか
- `tags` や `corners` フィールドに放送内容を詳述したテキストが入っていないか

## 3. 検索・索引の範囲か

このサイトが提供する情報が「検索・索引・導線」の範囲に収まっているか確認する。

許容される情報：
- 動画タイトル・放送回数・出演者・公開日
- コーナー名・楽曲タイトル（リクエスト曲名）
- イベント名・ライブ名（感想回の索引として）
- YouTube 動画への直接リンク

確認が必要な情報：
- YouTube 説明文の一部引用（短い補足情報として使う場合はタイトル・出演者情報の範囲内か確認）
- 誕生日・公開録音の実施概要（「〇〇の誕生日回」という索引情報は可、内容詳述は不可）

## 4. 権利表記があるか

ファンサイトとしての立場・著作権帰属が適切に表記されているか確認する。

チェック観点：
- サイト上に「ファンサイトである」旨の表記があるか、または表記が不要な範囲の情報のみを扱っているか
- 番組名・キャラクター名・ユニット名（Liella! 等）を使用する場合、公式の権利者表記（©）が適切に行われているか
- 新しくロゴ・アイコン・画像を追加する場合、自作または使用許諾があるものか（スクリーンショット・公式画像の無断使用は NG）
- PWA アイコンやサムネイル画像が著作権フリーまたは自作であるか

## 5. キャッシュしすぎていないか

外部データのキャッシュ期間が利用規約・著作権の観点で適切か確認する。

チェック観点：
- YouTube API レスポンスのキャッシュ TTL が **30 日以内**か（YouTube API ToS の上限）
- 動画メタデータ（タイトル・説明文・サムネイル URL）を長期間キャッシュしている場合、公式が削除・修正した際に古い情報を表示し続けないか
- Google Sheets のメタ情報キャッシュ TTL が適切か（現在: 1 時間）。長期キャッシュにする場合、陳腐化したデータを提供するリスクを検討したか
- `data/episodes.json` のフォールバックデータに、YouTube で削除済みの動画が含まれていないか

## 6. YouTube 規約に違反していないか

YouTube Data API・YouTube コンテンツの利用が利用規約の範囲内か確認する。

チェック観点：
- YouTube Data API から取得したデータを、API を経由せずに直接 JSON ファイルとして長期配布していないか（`data/episodes.json` は API 失敗時のフォールバックとしての位置づけを維持すること）
- サムネイル画像を YouTube の CDN（`i.ytimg.com`）から直接参照しているか（自サーバーに保存・再配信していないか）
- YouTube の埋め込みプレイヤーを使用する場合、`autoplay` や広告スキップなど規約違反のパラメータを使っていないか
- API キーをフロントエンドに露出させていないか（必ずサーバーサイドの `api/*.js` 経由でアクセスすること）
- 視聴回数・高評価数・コメントを取得・表示する場合、キャッシュ期間が適切か（リアルタイム性が求められるデータをキャッシュしすぎていないか）

---

チェック完了後、「公開可」「修正してから公開可」「公開不可（対処必須）」のいずれかを判定し、対処が必要な場合は優先度順に対処方法を示してください。
